# ASCII → 코드 핸드오프 패턴

ASCII 목업이 확정된 후 Claude Code / Codex / v0에 넘기는 구조화된 프롬프트를 자동 생성.

## 핸드오프 프롬프트 템플릿

목업 확정 시 아래 형식으로 자동 생성:

---

**[핸드오프 프롬프트]**

```
다음 ASCII 목업을 기반으로 [React/HTML/Vue] 컴포넌트를 만들어줘.

## 화면 정보
- 페이지명: [페이지명]
- 뷰포트: [웹 80자 / 모바일 40자]
- 선택된 안: [A안 / B안]

## ASCII 목업
[확정된 ASCII 목업 붙여넣기]

## 인터랙션 명세
[인터랙션 어노테이션이 있으면 그대로 포함]

## 콘텐츠 힌트
[Promptframe [AI: ...] 주석이 있으면 그대로 포함]

## 기술 스택
- 프레임워크: [React / Next.js / Vue]
- 스타일: [Tailwind / CSS Module / styled-components]
- 데이터: [props / API 연동 / 하드코딩 목업]

## 제약
- 컴포넌트 분리 기준: [영역별 / 페이지 단일]
- 반응형: [필요 / 불필요]
- 접근성: [aria 필요 / 불필요]
```

## 실제 예시 — hire 대시보드 핸드오프

```
다음 ASCII 목업을 기반으로 React + Tailwind 컴포넌트를 만들어줘.

## 화면 정보
- 페이지명: 채용 현황 대시보드
- 뷰포트: 웹 (80자 기준)
- 선택된 안: A안 (카드 리스트형)

## ASCII 목업
┌──────────────────────────────────────────────────────────────────────────────┐
│ 채용 현황                              [필터▼]①        [*+ 추가*]②          │
├──────────────────────────────────────────────────────────────────────────────┤
│ [*진행중 (3)*] [전체] [완료] [보류]  ③                                       │
├──────────────────────────────────────────────────────────────────────────────┤
│ 쿠팡페이    STEP 3 · 전략확정    D-3    [좋음] 80%    [*다음 →*]④           │
│ 아이오로라  STEP 4 · 포트폴리오  D-7    [주의] 60%    [*다음 →*]             │
│ 여기어때    STEP 2 · 서비스분석  D-14   [좋음] 75%    [*다음 →*]             │
└──────────────────────────────────────────────────────────────────────────────┘

## 인터랙션 명세
① 필터: 단계 / 우선순위 / 마감일 드롭다운. 선택 시 뱃지 표시
② 회사 추가 모달 오픈 → 회사명 + JD URL 입력 → state JSON 저장
③ 탭 전환 시 ?tab= URL 파라미터. 새로고침 유지
④ 클릭 시 파이프라인 다음 단계 실행. 실행 중 버튼 비활성화

## 기술 스택
- 프레임워크: React + Next.js
- 스타일: Tailwind CSS
- 데이터: props (상위에서 companies[] 배열 주입)

## 제약
- HireCard 컴포넌트 분리
- 모바일 반응형 불필요 (웹 전용)
```

## 언제 사용
