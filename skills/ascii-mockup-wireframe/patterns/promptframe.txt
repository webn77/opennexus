# Promptframe 패턴 (NN/G 방식)

빈 박스 대신 [AI: ...] 콘텐츠 힌트를 넣어 콘텐츠 의도를 명확히 전달.
팀/개발자가 목업만 봐도 각 영역에 무엇이 들어가야 하는지 알 수 있음.

## 표기 규칙

[AI: 내용 목표 + 데이터 출처 + 강조 조건]

- 내용 목표: 이 영역이 달성해야 할 커뮤니케이션 목적
- 데이터 출처: 어디서 가져오는 값인지
- 강조 조건: 특정 상황에서 다르게 표시할 규칙

## Promptframe 예시 — hire 대시보드

=== 채용 현황 — Promptframe ===

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ [AI: 사용자 이름 + "안녕하세요" 인사. 현재 진행 중인 회사 수 강조]           │
│                                              [필터▼]①   [*+ 추가*]②         │
├──────────────────────────────────────────────────────────────────────────────┤
│ [AI: 전체/진행중/완료/보류 탭. 각 탭에 건수 뱃지 표시. 기본은 진행중]       │
├──────────────────────────────────────────────────────────────────────────────┤
│ [AI: 마감 임박 TOP 3 카드. D-day 오름차순 정렬. D-3 이하면 빨간색 강조]     │
│                                                                              │
│ [AI: 각 카드에 회사명·현재 STEP 라벨·fit% 표시. fit 80% 이상이면 [좋음]]   │
│                                                                              │
│ [AI: 다음 액션 버튼. STEP별로 라벨 다름 — "서비스분석 시작" / "전략 확정"]  │
├──────────────────────────────────────────────────────────────────────────────┤
│ [AI: 이번 주 완료한 STEP 수 + 격려 메시지. 0건이면 "이번 주 첫 시작해요!"] │
└──────────────────────────────────────────────────────────────────────────────┘
```

## Promptframe + ASCII 혼합 (실용 버전)

구조는 ASCII로, 콘텐츠 의도만 [AI: ...]로 표기.

```text
=== 채용 현황 — Promptframe + 구조 ===

┌──────────────────────────────────────────────────────────────────────────────┐
│  채용 현황                             [필터▼]           [*+ 추가*]          │
├──────────────────────────────────────────────────────────────────────────────┤
│  [*진행중 (N)*]  [전체]  [완료]  [보류]                                      │
│  [AI: N = state JSON에서 status=active 건수 실시간]                          │
├──────────────────────────────────────────────────────────────────────────────┤
│  쿠팡페이     STEP 3 · 전략확정    D-3   ▲ 80%    [*다음 →*]                │
│  [AI: D-day는 applied_at+목표기간 계산. fit%는 STEP 1 결과값]               │
│                                                                              │
│  아이오로라   STEP 4 · 포트폴리오  D-7   ▲ 60%    [*다음 →*]                │
│  여기어때     STEP 2 · 서비스분석  D-14  ▲ 75%    [*다음 →*]                │
├──────────────────────────────────────────────────────────────────────────────┤
│  [AI: 주간 요약 — 이번 주 완료 STEP N건. 다음 마감까지 D-N일]               │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 언제 사용

- 개발자 핸드오프 전: 각 영역 데이터 출처와 조건 명확화
- 사용자 테스트 전: 실제 콘텐츠 의도 전달
- PRD 보완: 화면 목업에 콘텐츠 스펙 통합
- "어떤 데이터가 들어가야 해?" 질문이 반복될 때

트리거: "콘텐츠 의도도", "데이터 출처 포함해서", "promptframe으로" 언급 시 자동 적용.
