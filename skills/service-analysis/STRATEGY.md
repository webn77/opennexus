# 차별화 전략 분석

## STEP 5.5 — 차별화 전략 심층 분석

기존 STEP 5 인사이트에 추가되는 전략 프레임워크.
"왜 이 서비스가 존재하는가"와 "왜 이겨야 하는가"를 명확히 한다.

---

## 1. 서비스 존재 목적 (Why This Exists)

### Jobs-to-be-Done 분석
사용자가 이 서비스를 "고용"하는 이유:

```
[ 기능적 Job ]
사용자가 이 서비스로 해결하려는 실제 과제
예: "자료를 빠르게 찾고 싶다"

[ 사회적 Job ]
이 서비스를 씀으로써 보이고 싶은 모습
예: "일 잘하는 사람으로 보이고 싶다"

[ 감정적 Job ]
이 서비스를 쓸 때 느끼고 싶은 감정
예: "놓치는 것 없이 관리하고 있다는 안도감"
```

→ **핵심 Job 한 줄 요약**: [이 서비스가 대신해주는 것]

---

## 2. 차별화 전략 유형 진단

아래 4가지 중 이 서비스가 어디에 포지셔닝하는지 판단:

```
[원가 우위]     더 싸게 제공 → 가격 경쟁력
[차별화]        더 다르게 제공 → 특정 고객에게 더 나은 가치
[집중화]        더 좁게 제공 → 특정 세그먼트 완전 지배
[플랫폼/네트워크] 연결 효과 → 사용자가 많을수록 더 좋아짐
```

**현재 전략 진단 결과:** [해당 유형 + 근거 1~2줄]
**권고 전략 방향:** [현재 전략의 강화 or 전환 방향]

---

## 3. 블루오션 캔버스 (경쟁 차별화 지도)

업계가 당연하게 여기는 요소들 vs 이 서비스만의 독자 요소를 시각화.

```
[ 제거 (Eliminate) ]
업계 표준이지만 이 서비스가 없애버린 것
→ 이것을 없앤 이유와 고객 반응

[ 감소 (Reduce) ]
업계 수준보다 낮추거나 단순화한 것
→ 어떤 복잡성을 제거했는가

[ 증가 (Raise) ]
업계 평균보다 대폭 높인 것
→ 어느 가치를 극단적으로 높였는가

[ 창조 (Create) ]
업계에 없던 새로운 가치 요소
→ 이 서비스만이 만들어낸 새로운 게임의 규칙
```

---

## 4. 경쟁 우위 지속성 (Moat 분석)

이 서비스의 해자(Moat)가 얼마나 깊은가:

| Moat 유형 | 설명 | 현재 수준 | 강화 방법 |
|----------|------|---------|---------|
| 네트워크 효과 | 사용자가 많을수록 가치 증가 | 없음/약/중/강 | |
| 전환 비용 | 다른 서비스로 옮기기 어려운 정도 | 없음/약/중/강 | |
| 데이터 우위 | 축적된 데이터로 경쟁사가 따라올 수 없는 수준 | 없음/약/중/강 | |
| 브랜드 | 사용자 인식·신뢰 | 없음/약/중/강 | |
| 원가 구조 | 규모의 경제 또는 독보적 비용 효율 | 없음/약/중/강 | |
| 특허·IP | 법적 보호 | 없음/약/중/강 | |

**종합 Moat 점수:** /10
**가장 취약한 Moat:** [유형] → [강화 전략]

---

## 5. 차별화 전략 한 줄 요약 (Positioning Statement)

```
[서비스명]은 [타겟 고객]을 위한 [카테고리]로,
[기존 대안]과 달리 [핵심 차별점] 때문에
[핵심 혜택]을 제공한다.
```

예시:
> "Notion은 지식 노동자를 위한 올인원 워크스페이스로,
> Word/Excel/Confluence와 달리 블록 기반 유연성 때문에
> 하나의 도구로 문서·데이터베이스·프로젝트를 통합 관리할 수 있다."
